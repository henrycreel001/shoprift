# Content Modes

The skill operates in one of two modes. **Building-in-public is the default.** Switch to launch only when the user explicitly says so.

---

## Mode 1: Building in Public (DEFAULT — today)

**What's true right now**:
- The Shoprift web app does NOT exist yet. It's being built.
- Mayank did manual migrations while dm2buy was down. Those are real.
- dm2buy is back up. The framing is "migrate and scale," not "outage rescue."
- Pricing is not finalized.

**You CAN claim**:
- "I'm building Shoprift" / "We're building Shoprift"
- "I migrated [N] stores manually last month" (if true)
- "Working on the dm2buy → Shopify CSV export right now"
- "Early access — DM me to get on the list"
- "Waitlist is open" (if there's a waitlist)
- "Demo coming soon" (only if there's a real timeline)

**You CANNOT claim**:
- "Shoprift moves your store in 3 minutes" (as if the product exists and works that way today)
- "Try Shoprift now" / "Sign up at shoprift.io"
- "Used by [X] sellers" (unless that's real)
- Pricing claims ("₹X for X products") until pricing is set
- Any specific product feature as if it's live

**What this looks like in copy**:

✓ "I'm building a dm2buy → Shopify migration tool. Currently in waitlist mode — DM if you want early access."

✓ "Last month I migrated 12 dm2buy stores by hand. Now I'm turning that process into a tool. Here's what I learned…"

✓ "Shoprift is what I'm building. The goal: drop in a dm2buy URL, get a Shopify-ready CSV in minutes. Not live yet. Soon."

✗ "Shoprift migrates your store in 3 minutes. Try it now."

✗ "Hundreds of sellers have already used Shoprift."

---

## Mode 2: Launch / Live (FUTURE — flip switch when web app is live)

**Trigger to switch**: User explicitly says "we're live now" or "Shoprift is shipped" or similar.

**What changes**:
- Product claims in present tense are OK ("Shoprift moves your store in minutes")
- CTAs become "Start migration" / "Try it now" / "Migrate now"
- Pricing can be referenced if set
- Real user stories / testimonials can be featured

**What stays the same**:
- Voice rules (still no exclamation marks, no banned words, no fluff)
- All four pillars still apply
- Founder voice on @mayankmalikx still present-tense building-in-public for the ongoing work

---

## Hybrid messaging

There's a useful in-between: "soft launch" / "early access." Even in building-in-public mode, you can drive signups:

- ✓ "Early access list is open. First 50 sellers get migrated free. DM 'migrate'."
- ✓ "Waitlist: shoprift.io"
- ✓ "Looking for 5 sellers to test the migration this week. Free. DM me."

These create urgency without claiming the product is live.

---

## How to handle "when is it live?" enquiries

In DMs:

> "Building it now. I'm taking on a handful of manual migrations in the meantime — same output, same CSV, just done by me. If you want one of those slots, send the dm2buy URL."

This converts the waiting period into revenue + research. Honest, useful, doesn't fake urgency.

---

## The honesty test

Before publishing, ask: **If a dm2buy seller signed up because of this post, would the reality match the promise?**

If the post implies a button they can click to migrate today and that button doesn't exist, the answer is no. Rewrite to building-in-public mode.

This isn't just ethics. It's brand survival — the dm2buy seller community is small and tight. One "this guy lied about his product" message in a WhatsApp group ends the brand before it starts.
